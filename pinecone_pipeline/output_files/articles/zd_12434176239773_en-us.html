Fixing a 'non-standard path' error when adding a Bitcoin (BTC) account

This guide details how to resolve a 'non-standard path: 44'/145'/0'/0/0' error, occasionally displayed as UNKNOWN_ERROR (0x6a82), that you may encounter while setting up a Bitcoin (BTC) account.

What does it mean?

This error message typically shows up if you've previously set up a Bitcoin (BTC) account using your Ledger device and a third-party wallet, following a derivation path that isn't compatible with Ledger Live.

Troubleshooting

  1. Open Ledger Live.
  2. Navigate to Settings > Experimental features and ensure the Extended account search option is turned off.



    Disabling this option will prevent this error message from appearing again when adding new Bitcoin accounts.

    If you're trying to add an account that isn't recognized by Ledger Live due to a non-standard derivation path, we recommend connecting your Ledger device to a third-party wallet, such as Electrum or Sparrow, that supports custom derivation paths.
